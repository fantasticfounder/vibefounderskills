# /discover-customers -- Full Customer Discovery

Understand who your app is actually for. Run the complete customer discovery workflow: find your target audience, map competitors, validate demand, prepare for interviews, and build hypothesis personas - all in one session.

## Invocation

```
/discover-customers AI-powered habit tracker for gym beginners
/discover-customers SaaS dashboard for freelancers tracking income
/discover-customers [paste your one-line product description]
```

## Workflow

### Step 1: Understand the Product and Current State

Ask:
- What does your app do? (one sentence)
- Who did you build it for? (who you had in mind)
- Have you launched? If yes, who's actually using it?
- Do you have any data? (analytics, sign-ups, user feedback, support tickets, reviews)
- What do you already know about your market? (competitors, communities, relevant threads)

Accept context from uploaded files, links, or conversation. If the user provides existing research, use it - don't override it with web search (per CONVENTIONS.md).

### Step 2: Find Target Audience (~15 minutes)

Apply the **find-target-audience** skill:

- Map 5-8 potential audience segments
- Score each on pain intensity, willingness to pay, reachability, market size, and founder-market fit (1-5 each)
- Build mini-ICPs for the top 2-3 segments using JTBD (Clayton Christensen, "Competing Against Luck", 2016)
- Recommend ONE primary target

**Checkpoint**: "Based on the analysis, **[segment]** scores highest as your primary target at [X/25]. Here's why: [rationale]. Does this feel right, or do you want to explore a different segment?"

If the user disagrees or wants to explore another segment, adjust. If they change direction, re-derive all downstream outputs from the new target (per CONVENTIONS.md context threading rules).

### Step 3: Scan Competitors (~10 minutes)

Apply the **competitor-scan** skill using the primary target audience:

- Map 5-7 competitors across direct, indirect, DIY, and "do nothing" categories
- Apply simplified Porter's Five Forces (Michael Porter, "Competitive Strategy", 1980)
- Build a value curve comparing the founder's product against top 3 alternatives (Blue Ocean Strategy, W. Chan Kim and Renee Mauborgne, 2004)
- Identify the gap where the founder's product wins

Carry the target audience from Step 2 into this analysis - scope competitors to the chosen segment, not the entire market.

### Step 4: Validate Demand (~10 minutes)

Apply the **demand-signals** skill:

- Check 5 signal sources: search signals, Reddit, Twitter/X, community signals, and alternative/payment signals
- Score each source as Strong / Moderate / Weak / None
- Calculate overall demand confidence: High / Moderate / Low
- Extract demand language - the exact phrases people use

**Checkpoint**: "Demand confidence is **[level]**. Here's what the signals show: [summary]. Should we continue building for this audience, or does this change your thinking?"

If demand confidence is Low, recommend going back to Step 2 to try a different audience segment before continuing.

### Step 5: Prepare Interview Script (~10 minutes)

Apply the **customer-interview-script** skill:

- Clarify what decision the interviews will inform
- Build a 4-section Mom Test script (Rob Fitzpatrick, "The Mom Test", 2013) tailored to the target audience and product
- Include inline Mom Test rules, probing techniques, and note-taking template

Use the target audience from Step 2 and the competitive insights from Step 3 to tailor the script. If the competitor scan revealed unexpected alternatives, include questions about those.

### Step 6: Build Hypothesis Personas (~10 minutes)

Apply the **user-personas** skill:

- Build 2-3 hypothesis personas based on the target audience analysis, competitor insights, and demand signals
- Use JTBD structure (Clayton Christensen, "Competing Against Luck", 2016)
- Rank by pain intensity, willingness to pay, and reachability
- **Mark ALL personas as UNVALIDATED** - interviews haven't happened yet

Carry forward ALL data from previous steps: segment scores, competitor gaps, demand language, and signal sources. The personas should reflect everything discovered so far.

### Step 8: Offer Next Steps

Based on the results:

**If demand confidence is High:**
- "Your audience is clear and demand is strong. Run 8-12 interviews to validate, then move to positioning. Use /ff-positioning:position when ready."

**If demand confidence is Moderate:**
- "There are signals, but they're not conclusive. Interviews are critical - they'll either confirm or redirect. Use the interview script above and run 8-12 conversations."

**If demand confidence is Low:**
- "The signals are weak for this audience. Before interviewing, consider: can you reframe the product for a different audience? Use find-target-audience again with a different angle, or use /ff-idea-validation:validate to check whether the core idea has demand."

## Notes

- This full workflow takes 60-90 minutes across multiple exchanges
- At each checkpoint, the user can redirect, skip ahead, or go deeper
- The report is designed to be saved and referenced throughout the product lifecycle
- Be honest throughout. A discovery process that confirms bad assumptions is worse than no process at all.
- Context threading is critical: every step builds on the previous one. Do not re-ask for information already gathered.
