# /grow -- Full Growth Strategy

Build a complete growth strategy from scratch: define your north star metric, diagnose retention gaps, design sustainable growth loops, build referral and email systems, and assess scaling readiness - all in one workflow.

## Invocation

```
/grow AI-powered invoice tool for freelancers
/grow habit tracking app with 200 users
/grow [paste your product description]
```

## Workflow

> Read CONVENTIONS.md for web search policy, context threading, and scoring conventions. These rules govern the entire workflow.

Carry forward outputs between steps. North star metric from Step 2 feeds into Step 3's retention analysis. Retention gaps from Step 3 inform growth loop design in Step 4. If the user changes direction at any checkpoint, re-derive all downstream steps.

### Step 1: Understand Current State

Ask:
- What does your product do? (one sentence)
- How many users do you have right now?
- What's your monthly revenue (if any)?
- How are users finding you today? (organic, referral, paid, community, etc.)
- What's your current growth rate? (new users per week/month)
- What's the biggest growth challenge you're facing?
- Do you have analytics set up? (PostHog, Mixpanel, Amplitude, or anything)
- What have you tried so far that hasn't worked?

Accept context from uploaded files, analytics screenshots, or conversation history.

### Step 2: Define Your North Star Metric (~10 min)

Apply the **north-star-metric** skill:

- Identify 3-5 candidate metrics based on the product's core value
- Test each against: measures value, leading indicator, actionable, simple, connected to revenue
- Select ONE North Star Metric with clear justification
- Define 3-4 input metrics that drive it
- Set realistic targets for current stage

**Checkpoint**: "Your North Star Metric is [metric]. Does this capture the core value your product delivers?"

### Step 3: Diagnose Retention (~15 min)

Apply the **retention-audit** skill:

- Assess data availability (analytics, feedback, support tickets)
- If no data: help set up minimum viable retention tracking
- If data exists: diagnose across 4 stages (onboarding, short-term, medium-term, long-term)
- Apply the Habit Loop (Nir Eyal "Hooked") to the product
- Identify the #1 retention lever
- Create a prioritised improvement plan

**Checkpoint**: "Your biggest retention gap is [stage]. Here's what I recommend fixing first."

### Step 4: Design Growth Loops (~15 min)

Apply the **growth-loops** skill:

- Evaluate 5 growth loop types for product fit
- Score each on: product fit, implementation effort, compound potential, founder resources
- Design mechanics for the top loop (trigger, incentive, friction, measurement)
- Build a 30-60-90 day implementation plan
- Identify a secondary loop to layer on later

### Step 5: Build a Referral System (~10 min)

Apply the **referral-system** skill:

- Determine if referrals make sense for this product
- Choose the referral type (product-led, incentivized, status-based, community-based)
- Design the incentive structure (two-sided preferred)
- Design the mechanics (sharing method, timing, tracking)
- Set realistic targets and implementation plan

### Step 6: Design Email List Strategy (~10 min)

Apply the **email-list-strategy** skill:

- Design the lead magnet (score options on relevance, effort, perceived value)
- Map email capture points
- Outline the 5-email welcome sequence
- Recommend ongoing strategy and tools

### Step 7: Assess Scaling Readiness (~10 min)

Apply the **scale-audit** skill:

- Map current time allocation (build, sell, support, operate)
- Apply leverage test to each activity (automate, eliminate, delegate, scale?)
- Project unit economics at 10x and 100x
- Build a scaling roadmap with revenue milestones
- Surface the lifestyle vs growth choice

## Notes

- Allow 90-120 minutes across multiple exchanges
- At each checkpoint, the user can redirect, skip, or go deeper
- The report is designed to be a living document - update it as the product grows
- If the user hasn't got any users yet, strongly recommend running /launch first
- Growth without retention is a leaky bucket - the retention audit should come before growth loops
- Be honest about what works for indie products. Most viral growth strategies require network effects that solo products don't have.
